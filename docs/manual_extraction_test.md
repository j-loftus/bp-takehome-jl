# Manual Extraction Test — Single Document

Run this before the full batch to validate prompt quality on a handful of documents.

## Prerequisites

```bash
export ANTHROPIC_API_KEY=sk-ant-...
```

Verify the sample manifest exists (generated by `scripts/sample_contracts.py`):

```bash
ls data/sample_manifest.csv
```

## Run the test

```bash
python -m src.pipeline.extractor --test path/to/document.pdf
```

Output is printed to stdout: parser summary, classifier result, extraction JSON, and token usage.

## Recommended test sequence

Cover at least three doc types before running the full batch. The highest-variance cases are:

| Doc type | What to look for |
|----------|-----------------|
| `fully_executed_agreement` | `total_contract_value` populated (from Exhibit A), all dates present, `price_escalator_terms` a valid enum |
| `renewal_letter` | `contract_start_date` / `contract_end_date` are the *renewal period* dates, not the original contract dates |
| `modification_amendment` | `modification_financial_delta` present if the amendment changes value; `parent_contract_number` extracted |

After those look clean, spot-check a `vendor_disclosure_statement` and an `award_letter`.

## What to watch for

- `extraction_status: failed` — check `failure_reason` to see whether the LLM returned bad JSON, an invalid enum, or a null non-nullable field
- `extraction_confidence: low` — the LLM flagged uncertainty; check `extraction_notes` for detail
- `total_contract_value: null` on a fully executed agreement — Exhibit A may be blank/redacted (expected), or the prompt may need tuning
- Dates in wrong format — the LLM returned something other than `YYYY-MM-DD`; treat as a prompt iteration signal
- `doc_type` in the extraction JSON doesn't match the classifier output — the LLM reclassified despite instructions; flag for prompt review

## Iterate on prompt issues

If results look wrong, edit [src/pipeline/prompts.py](../src/pipeline/prompts.py) and re-run. The extractor imports `build_extraction_prompt` fresh on each run — no restart needed.
